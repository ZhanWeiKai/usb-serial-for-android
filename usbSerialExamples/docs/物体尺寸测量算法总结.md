# ObjectMeasurementActivity 物体尺寸测量算法总结

## 概述

基于 A010 TOF 深度传感器的物体三维尺寸测量系统，通过两阶段流程（校准+测量）计算放置在桌面上物体的长、宽、高。

---

## 1. 采样方式

### 采样频率
```
采样周期: 500ms (0.5秒)
每周期内: 采样 10 帧原始深度图
输出: 取 10 帧中值，得到 1 个降噪深度图
页面刷新: 每 0.5 秒更新一次
```

### 中值降噪原理

```
每 0.5 秒周期:
┌─────────────────────────────────┐
│  采样 10 帧原始深度图            │
│  frame_1[100][100]              │
│  frame_2[100][100]              │
│  ...                            │
│  frame_10[100][100]             │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│  对每个像素位置取中值            │
│                                 │
│  medianDepth[x][y] = median(    │
│    frame_1[x][y],               │
│    frame_2[x][y],               │
│    ...                          │
│    frame_10[x][y]               │
│  )                              │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│  得到 1 个降噪深度图             │
│  medianDepth[100][100]          │
└─────────────────────────────────┘
```

### 中值降噪举例

```
假设某像素 (50,50) 在 10 帧中的深度值:
frame_1:  350mm
frame_2:  352mm
frame_3:  480mm  ← 异常值（噪声）
frame_4:  351mm
frame_5:  349mm
frame_6:  353mm
frame_7:  350mm
frame_8:  999mm  ← 异常值（噪声）
frame_9:  351mm
frame_10: 350mm

排序后: [349, 350, 350, 350, 351, 351, 352, 353, 480, 999]
中值 = 351mm  ← 自动剔除了噪声
```

**优势**: 中值滤波天然能剔除脉冲噪声，比平均值更鲁棒。

### 关键参数
| 参数 | 值 | 说明 |
|------|-----|------|
| `SAMPLE_INTERVAL_MS` | 500ms | 采样周期 |
| `FRAMES_PER_INTERVAL` | 10 | 每周期采样帧数 |
| `STABLE_COUNT_CALIBRATE` | 10 | 校准需连续 10 个周期稳定 = 5 秒 |
| `STABLE_COUNT_MEASURE` | 10 | 测量需连续 10 个周期稳定 = 5 秒 |
| `BASELINE_AVG_THRESHOLD_MM` | 14mm | 帧间差异阈值，小于此值才算稳定 |
| `OBJECT_THRESHOLD_MM` | 50mm | 物体检测阈值，深度差 > 50mm 才算物体 |

### 帧格式
```
包头: 2字节 (0x00, 0xFF)
包长度: 2字节 (小端)
元数据: 16字节
图像帧: 10000字节 (100×100 深度图)
校验: 1字节
包尾: 1字节 (0xDD)
```

## 2. 校准流程

### 目的
扫描**空桌面**，建立深度基准，计算像素-物理尺寸映射比例。

### 步骤
1. 点击"开始校准"，清空桌面
2. 每 0.5 秒：
   - 采样 10 帧原始深度图
   - 取中值得到 1 个降噪深度图
   - 与上一个降噪深度图比较差异
3. 差异 **< 14mm** 才算稳定帧
4. 累积 **10 个稳定周期**后，**取中位数**作为最终基线

### 校准输出
```java
baselineDepth[100][100]  // 空桌面深度矩阵 (mm) - 10个降噪帧的中位数
pixelSizeX               // X方向每像素对应的 mm
pixelSizeY               // Y方向每像素对应的 mm
```

### 像素尺寸计算公式
```
pixelSizeX = 2 × depth × tan(FOV_H/2) / 100
pixelSizeY = 2 × depth × tan(FOV_V/2) / 100

其中:
- FOV_H = 70° (水平视场角)
- FOV_V = 60° (垂直视场角)
- depth = 中心点深度 (mm)
- 100 = 图像像素尺寸
```

### 为什么用中位数而不是平均值？

中位数对异常值（噪声）更鲁棒，能有效剔除偶然的深度测量错误。

## 3. 确定像素范围（物体检测）

### 3.1 生成物体 Mask

使用降噪后的深度图，通过深度差异检测物体像素：

```java
depthDiff = baselineDepth[x][y] - currentMedianDepth[x][y]

if (depthDiff > 50mm) {
    mask[x][y] = 1;  // 物体像素
} else {
    mask[x][y] = 0;  // 背景像素
}
```

**原理**: 有物体时，该位置深度变小（传感器到物体表面的距离 < 到桌面的距离）

### 3.2 连续像素法确定边界

不使用梯度法，改用**连续像素计数法**：

```java
// 行方向: 统计每行有多少个连续的 mask=1 像素
for (int y = 0; y < 100; y++) {
    int consecutiveCount = 0;
    for (int x = 0; x < 100; x++) {
        if (mask[x][y] == 1) {
            consecutiveCount++;
        } else {
            if (consecutiveCount >= 8) {
                // 该行有效，记录到有效行集合
                validRows.add(y);
            }
            consecutiveCount = 0;
        }
    }
    // 行末检查
    if (consecutiveCount >= 8) {
        validRows.add(y);
    }
}

// 列方向: 同理统计每列的连续像素数
for (int x = 0; x < 100; x++) {
    int consecutiveCount = 0;
    for (int y = 0; y < 100; y++) {
        if (mask[x][y] == 1) {
            consecutiveCount++;
        } else {
            if (consecutiveCount >= 8) {
                validCols.add(x);
            }
            consecutiveCount = 0;
        }
    }
    if (consecutiveCount >= 8) {
        validCols.add(x);
    }
}
```

### 3.3 边界确定规则

| 方向 | 条件 | 输出 |
|------|------|------|
| 行 (Y) | 连续 **8 个** mask=1 像素 | 该行有效 → 确定 yMin, yMax |
| 列 (X) | 连续 **8 个** mask=1 像素 | 该列有效 → 确定 xMin, xMax |

**最终边界**: `xMin = min(validCols)`, `xMax = max(validCols)`, `yMin = min(validRows)`, `yMax = max(validRows)`

## 4. 计算高度

### 方法: 深度差异法

```java
// 找最大深度差（使用降噪后的深度图）
maxDepthDiff = max( baselineDepth[x][y] - currentMedianDepth[x][y] )

// 传感器45°倾斜，投影到垂直方向
height = maxDepthDiff × cos(45°)
       = maxDepthDiff × 0.707
```

### 原理图解

```
        传感器
          ↘ 45°
            \
             \  斜向深度差 (maxDepthDiff)
              \
               ↓
           ┌───────┐
           │ 物体  │  ← 高度 H
           │       │
           └───────┘
    ───────────────────
          桌面

几何关系:
maxDepthDiff = H / cos(45°) = H × √2
H = maxDepthDiff × cos(45°) = maxDepthDiff × 0.707
```

---

## 5. 计算长和宽

### 方法: 先算高度，再动态计算像素尺寸，最后算长宽

**重要**: 像素尺寸 pixelSizeX/Y 必须基于**物体表面深度**计算，而不是桌面深度。

```java
// ========== 步骤1: 先计算高度 ==========
maxDepthDiff = max( baselineDepth[x][y] - currentMedianDepth[x][y] )
高度 = maxDepthDiff × cos(45°)

// ========== 步骤2: 计算物体表面深度 ==========
// 物体表面深度 = 基线深度 - 斜向深度差
// 也就是 currentMedianDepth（传感器到物体表面的距离）
物体表面深度 = baselineDepth - maxDepthDiff

// 或者直接用当前测量的深度
物体表面深度 = currentMedianDepth[x_maxDiff][y_maxDiff]  // 最大深度差位置的深度

// ========== 步骤3: 基于物体表面深度动态计算像素尺寸 ==========
// 物体表面越近，每像素对应的物理尺寸越小
pixelSizeX = 2 × 物体表面深度 × tan(35°) / 100  // 35° = FOV_H/2
pixelSizeY = 2 × 物体表面深度 × tan(30°) / 100  // 30° = FOV_V/2

// ========== 步骤4: 计算长宽 ==========
xSpan = xMax - xMin + 1  // X方向像素跨度
ySpan = yMax - yMin + 1  // Y方向像素跨度

宽  = xSpan × pixelSizeX
长 = ySpan × pixelSizeY
```

### 为什么必须用物体表面深度？

```
传感器视角 (45°斜向下看):
                传感器
                  ↘ 45°
                    │
         ┌──────────┼──────────┐
         │    桌面   │          │  ← 桌面深度 ~400mm
         │          │    ↑     │     每像素 = 2×400×tan(35°)/100 ≈ 5.6mm
         │          │  高度H   │
         │          │    ↓     │
         │    物体  │          │  ← 物体表面深度 ~350mm
         └──────────┴──────────┘     每像素 = 2×350×tan(35°)/100 ≈ 4.9mm

如果用桌面深度(400mm)计算像素尺寸:
  → 会高估物体的长宽（用更大的像素尺寸乘以像素数）

正确做法: 用物体表面深度(350mm)计算像素尺寸
  → 得到准确的长宽
```

**结论**: 物体表面离传感器越近，该平面上每像素对应的物理尺寸越小。

## 6. 传感器 45° 斜向下的处理

### 坐标系定义

```
相机坐标系 (传感器视角):
- X_cam: 图像水平方向 (右为正)
- Y_cam: 图像垂直方向 (下为正)
- Z_cam: 深度方向 (远为正)

世界坐标系 (桌面视角):
- X_world: 平行于桌面，与 X_cam 相同
- Y_world: 平行于桌面，向远为正
- Z_world: 垂直于桌面向上
```

### 45° 旋转变换矩阵

绕 X 轴旋转 45°，将相机坐标系转换到世界坐标系：

```
| X_world |   | 1    0        0     |   | X_cam |
| Y_world | = | 0  cos(45°) -sin(45°) | × | Y_cam |
| Z_world |   | 0  sin(45°)  cos(45°) |   | Z_cam |

简化 (cos(45°) = sin(45°) = 0.707):
X_world = X_cam
Y_world = 0.707 × Y_cam - 0.707 × Z_cam
Z_world = 0.707 × Y_cam + 0.707 × Z_cam
```

### 实际应用

代码中使用简化方法：
- **高度**: 直接用 `maxDepthDiff × cos(45°)` 修正
- **长宽**: 通过 FOV 公式自动考虑透视关系

## 算法要点总结

1. **采样降噪**: 每 0.5 秒采样 10 帧，对每个像素取中值，得到 1 个降噪深度图
2. **校准**: 10 个稳定周期（差异<14mm）后，取中位数作为基线
3. **Mask生成**: 深度差 > 50mm 的像素标记为物体
4. **边界检测**: 连续 ≥8 个 mask=1 像素的行/列才算有效
5. **先算高度**: maxDepthDiff × cos(45°)
6. **再算像素尺寸**: 基于物体表面深度 = baselineDepth - maxDepthDiff
   - pixelSizeX = 2 × 物体表面深度 × tan(35°) / 100
   - pixelSizeY = 2 × 物体表面深度 × tan(30°) / 100
7. **最后算长宽**: 边界跨度 × 动态像素尺寸
8. **页面刷新**: 每 0.5 秒展示一次测量结果
